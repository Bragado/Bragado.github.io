AR for Medical Care

The aim of this project is to develop a mobile iOS and Android application in React Native that allows nurses, when entering a hospital room, to identify a set of patterns and replace them with an augmented reality model with relevant information for the nurse. This model must be fed by a REST service that should indicate the type of model to generate as well as the information necessary for its completion. The models should allow rapid access to the intervention registration module and/or other actions considered relevant (such as access to the patient's process). The area for registration of interventions should be provided by the mobile application and the information collected should be sent to an FHIR service, following the CDA v2 structure, so that it can be easily shared by the different hospital systems concerned.

Alexa

This project aims to treat information and integrate with the internal MES of the client’s company. It has as focus the development of a chat bot which will process and process information so that proper communication with the MES system is possible. The introduction of data by the user will be done through terminals such as Skype, Alexa and Microsoft Teams. Communication between the different layers will be done through HTTP. In the future authentication and access data to credible MES information will be provided to better conduct tests and plan the project in a real context.

Speech Recognition

The aim of this project is "Speech Recognition and Mood Classifier" with intention of assessing the state of mind of a call between an agent and a contact. The evaluation criteria are given by the Talkdesk clients, having access to a Backoffice where contact centers supervisors can enter keywords which are classified as negative or positive. After a call the conversation is converted into text and analyzed by the previously defined keywords, thus presenting a positive or negative result of the call. In addition, the team is required to perform a market study, to discover the value this product has for a customer and a characterization of the target customers for this product.

Serious Games

The aim of this project is the development of a mobile application aimed at health professionals, parents and children aged 3 to 6 years who are overweight and without chronic diseases or already defined food plan. The mobile application has three dimensions: multimedia content for parent’s awareness and encouragement of good food practices; a serious game for the children and an area for a health professional (usually the family doctor) to supervise the progress of both parents and children. Due to a third party being responsible for creating the serious game, the team’s job is to interconnect the various components and create the brand and image of the application.
